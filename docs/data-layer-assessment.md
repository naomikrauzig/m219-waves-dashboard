# M219 WAVES Cruise Dashboard Data Layer Assessment

## A. Modeled Forecast

Status: operational in the current GitHub Actions workflow.

The dashboard can automatically fetch a rolling window from today through five days ahead for forecast-capable Copernicus Marine products. Current automated layers are:

- Wave height and period from `GLOBAL_ANALYSISFORECAST_WAV_001_027`
- Ocean currents from `GLOBAL_ANALYSISFORECAST_PHY_001_024`
- Temperature from `GLOBAL_ANALYSISFORECAST_PHY_001_024`
- Salinity from `GLOBAL_ANALYSISFORECAST_PHY_001_024`

These layers are suitable for the daily forecast-map selector because they are model analysis/forecast products.

## B. Satellite-Derived Maps

Status: feasible, but not true forecasts.

Satellite products can be added for recent available days and cruise nowcasting:

- Sea level anomaly, absolute dynamic topography, and geostrophic currents
- Sea surface temperature
- Chlorophyll-a and ocean colour products

Limitations:

- Satellite maps are observation products, so they lag real time and cannot fill a five-day future window.
- SST and chlorophyll can be affected by cloud cover and quality masks.
- The page should show these in the same dropdown, but availability should be date-dependent.

## C. Argo Float And Drifter Data

Status: feasible as overlays, not gridded maps.

Argo profiles and Argo-derived velocities can be queried from Copernicus Marine in-situ products. NOAA Global Drifter Program data are freely available through public data services including ERDDAP.

Recommended display:

- Float and drifter positions on the route map
- Trajectories over the selected map date range
- Profile popups for temperature/salinity versus pressure or depth

Limitations:

- These data are sparse and unevenly distributed along the cruise route.
- They should be plotted as points/lines over maps rather than treated as gridded fields.

## D. Derived Variables

Status: feasible with additional compute routines.

Useful derived products for M219 could include:

- Ekman pumping from wind-stress curl: `w_E = curl(tau) / (rho * f)`
- Upwelling proxies from winds, SST gradients, and surface currents
- Eddy kinetic energy from geostrophic currents: `EKE = 0.5 * (ugos^2 + vgos^2)`
- Eddy masks or diagnostics from SLA/ADT using Okubo-Weiss or closed-contour methods

Limitations:

- Derived fields need consistent grids, units, and masks.
- Near the equator, Ekman diagnostics require care because the Coriolis parameter approaches zero.
- Eddy diagnostics are more robust from satellite altimetry than from short forecast windows.

## Dashboard Interaction

The current dashboard supports the requested interaction pattern:

- Dataset selection by dropdown
- Day selection by dropdown
- Availability table for available maps per date
- Real maps when generated by the workflow, placeholders otherwise

The next implementation step is adding separate processing scripts for satellite nowcasts, in-situ overlays, and derived diagnostics.
